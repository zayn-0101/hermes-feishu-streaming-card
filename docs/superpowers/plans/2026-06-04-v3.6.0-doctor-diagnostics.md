# V3.6.0 Doctor Diagnostics Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add read-only operational diagnostics for V3.6.0 with `doctor --json` and `doctor --explain`.

**Architecture:** Keep the existing plain `doctor` output stable, and add a shared diagnostics report builder for the new output modes. The report should summarize config loading, sidecar address, Hermes detection, streaming config, install manifest/backup state, and actionable recommendations without modifying any Hermes files.

**Tech Stack:** Python argparse, JSON, pytest, existing Hermes fixture and installer safety helpers.

**Out of Scope:** Do not implement `repair`, `setup --repair`, media/file delivery, Docker packaging, or write operations against a user Hermes install in this task.
